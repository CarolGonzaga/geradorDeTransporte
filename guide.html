Aba 1: Pacote e iFlow

Item 1: Advanced Event Mesh (AEM)

Marque 'Sim' se sua integração utilizar um AEM.

Atenção: Para cada integração com AEM, você deverá gerar três planilhas separadas:
  • Uma para o iFlow principal.
  • Uma para a Fila Principal do AEM.
  • Uma para a Fila Morta (DMQ) do AEM.

Item 3: Pacote já está em QAS?

Primeiro, verifique no ambiente de QAS se o pacote da sua integração já existe.

  • Se o pacote já existir em QAS, mantenha a opção 'Sim'.
  • Se o pacote não existir em QAS, marque a opção 'Não' e anexe uma evidência que mostre o pacote no ambiente de desenvolvimento.

Imagem 1: Pacote visível no ambiente de desenvolvimento.

Item 4: Nome do iFlow e Evidência

Informe o nome exato do iFlow que será transportado e anexe a evidência da sua versão.

Pontos de Atenção:
  • Apenas um iFlow por planilha.
  • O status do iFlow deve ser 'STARTED'.
  • O iFlow deve ter uma versão salva. Artefatos em modo 'Draft' não são transportados.

4.1 Evidência do iFlow: Anexe uma imagem que mostre claramente o nome do iFlow e sua versão mais recente.

Aba 2: API Management

Item 5: Dados da API Management

Se a sua integração utiliza API Management, marque 'Sim' e preencha os quatro sub-itens obrigatórios: API Proxy, Product, Application e Key Value Maps (KVM).

Para cada um deles, informe o nome exato e anexe uma evidência correspondente.

Aba 3: Credenciais e Roles

Item 7: Usuário SAP para Credencial

Informe o nome da Credential Name configurada dentro do iFlow. Este é o usuário técnico que será usado para a comunicação com outros sistemas.

Item 8: Ambiente SAP do usuário

Informe o ambiente SAP de destino. O valor padrão é NC2CLNT210, mas você deve alterá-lo caso a integração se comunique com um sistema diferente (ex: S4QCLNT210).

Item 9: User Role

Informe a User Role utilizada no iFlow para controle de acesso. O campo já vem preenchido com base no nome do pacote, mas pode ser alterado se a sua configuração for diferente.

Aba 4: Configure (Externalizações)

Item 11: ID do Delta

Preencha com o ID do Delta (Ex: PTM3772, OTC4051). Este ID será usado para nomear o arquivo Excel final, facilitando a organização e o rastreamento.
